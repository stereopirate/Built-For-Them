# Built For Them — Project Tracker

Project tracking (active queue, decisions, research log, ideas backlog) is maintained in an internal spreadsheet. The link is intentionally not committed to this public repo.

## What's in the tracker

| Tab | Contents |
|-----|----------|
| Projects | All BFT-### projects with status, priority, and scoring |
| Ideas | IDEA-### backlog with disposition |
| Active Queue | Current 3-slot work queue with blockers and next actions |
| Components | Shared COMP-### components reused across projects |
| Decisions | DEC-### log with rationale and revisit triggers |
| Research | RES-### citations and key findings |
| Reference | Scoring rubric and status definitions |

## Session start protocol

1. Fetch the internal tracker (link kept outside this public repo)
2. Note active queue slots and current blockers from the Active Queue tab
3. Cross-reference with `DECISIONS.md` and `ORIGINALS.md` for any gaps
4. Flag conflicts between tracker and project files before starting work
