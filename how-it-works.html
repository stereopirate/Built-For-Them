How it works

From real need to practical build.

Built For Them documents the full path—not just the polished final photo.

Define the need

Start with a specific activity, obstacle, environment, and set of functional requirements.

Prototype carefully

Create a simple version, review it with appropriate professionals, and test cautiously under supervision.

Publish what works

Share dimensions, materials, tools, limitations, safety notes, and honest testing status.

Project status matters

Every page uses a visible development label.

  • Early idea: a need and possible solution, not yet designed.
  • Concept: a defined design direction, not yet physically tested.
  • Prototype: a physical build exists, but testing is limited.
  • Field tested: used repeatedly with documented feedback.
  • Published plan: instructions and files have completed review.
